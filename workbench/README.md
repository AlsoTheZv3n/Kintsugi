# workbench/

Die Incident-Workbench. **Wird erst in Phase 4 gebaut** — siehe
[`docs/08-roadmap.md`](../docs/08-roadmap.md), Abschnitt „Phase 4".

Begründung für den späten Zeitpunkt, aus `docs/08-roadmap.md`:

> Sie ist das aufwendigste Stück Frontend und wertlos, solange es keine echten Incidents mit
> echter Evidence gibt. Zuerst muss das System Vorfälle produzieren, die es zu bearbeiten lohnt.

Was sie leisten muss, steht in [`docs/06-observability.md`](../docs/06-observability.md),
Abschnitt „Incident-Workbench": Beweislage, DOM-Diff, Selektor-Editor mit Sofortauswertung,
Ein-Klick-Fixture-Replay und die Aktion „als Fehlalarm schließen", die einen Negativtest im
Mutations-Harness erzeugt.

Dieses Verzeichnis existiert vorerst nur, damit die Repo-Layout-Tabelle in der README der
Wirklichkeit entspricht.
